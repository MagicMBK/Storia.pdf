1. INTRODUZIONE

L'obiettivo del progetto è la realizzazione dell'infrastruttura di rete interna dell'azienda WeDev S.r.l., finalizzata alla centralizzazione della gestione di utenti, computer e servizi di rete.

La soluzione adottata si basa sull'implementazione di un'architettura Windows Server, capace di garantire gestione centralizzata, sicurezza potenziata ed efficienza operativa ottimizzata.

Il progetto riguarda una rete aziendale locale (LAN) costituita da circa 25 postazioni client Windows, dislocate nelle diverse aree funzionali dell'azienda.

L'attuale gestione decentralizzata delle postazioni genera criticità nel controllo degli accessi, problematiche di sicurezza e complessità nella manutenzione ordinaria.

L'adozione di un dominio Windows permette di ottenere:

  • autenticazione centralizzata degli utenti
  • gestione uniforme e semplificata dei computer
  • controllo granulare degli accessi alle risorse
  • incremento della sicurezza complessiva e scalabilità dell'infrastruttura

2. ANALISI DEI REQUISITI

Contesto Operativo e Utenti

WeDev S.r.l. gestisce 25 postazioni client per personale amministrativo, sviluppatori e tecnici IT. Gli utenti accederanno tramite credenziali di dominio wedev.local per autenticarsi, utilizzare risorse condivise, consultare l'intranet e connettersi da remoto (se autorizzati).

Ogni postazione sarà configurata dall'amministratore e associata al dominio. I permessi saranno gestiti tramite Unità Organizzative (OU) e gruppi di sicurezza, con policy di scadenza password ogni 90 giorni. I dati risiedono su storage centralizzato con backup periodici e configurazione RAID per garantire continuità operativa.

Requisiti Hardware e Software

Hardware

  • Server dedicato (16-32 GB RAM, RAID) per AD DS, DNS, IIS e Desktop Remoto
  • Router/Firewall con DHCP e protezione perimetrale
  • 25 PC client Windows
  • Switch in rack protetti con patch panel
  • Cablaggio Ethernet strutturato

Software

  • Windows Server 2019/2022
  • Windows 10/11 client
  • Active Directory Domain Services
  • DNS Server integrato
  • IIS Web Server
  • Remote Desktop Services

Requisiti Funzionali e Non Funzionali

Funzionali: autenticazione centralizzata, gestione computer tramite dominio, risoluzione DNS, hosting intranet, accesso remoto sicuro, condivisione controllata risorse.

Non Funzionali:

  • Prestazioni: tempi di risposta rapidi per autenticazione e servizi
  • Sicurezza: firewall, policy di gruppo, permessi NTFS, audit
  • Scalabilità: espansione fino a 50+ utenti senza modifiche strutturali
  • Affidabilità: backup automatici, RAID, uptime elevato
  • Usabilità: gestione centralizzata tramite console amministrative

3. PROGETTAZIONE

Progettazione Concettuale

La rete di WeDev S.r.l. è progettata partendo da una rete privata di classe C 192.168.1.0/24, che fornisce un numero adeguato di indirizzi IP per le 25 postazioni client e i servizi server necessari.

La topologia prevede:

  • Un server Windows che funge da Domain Controller, DNS e Web Server
  • Client Windows collegati tramite switch
  • Router/Gateway per la connettività Internet
  • Servizi erogati: AD DS, DNS, IIS, Desktop Remoto

Progettazione Logica

Piano di Indirizzamento IP

L'assegnazione degli indirizzi IP avviene in modalità statica per il server e per i client, garantendo stabilità dei servizi critici e facilità di gestione della rete aziendale di piccole dimensioni.

Dispositivo Indirizzo IP Subnet Mask Gateway DNS
Server WeDev 192.168.1.10 255.255.255.0 192.168.1.1 192.168.1.10
PC-Client-01 192.168.1.20 255.255.255.0 192.168.1.1 192.168.1.10
PC-Client-02 192.168.1.21 255.255.255.0 192.168.1.1 192.168.1.10
Router/Gateway 192.168.1.1 255.255.255.0 - -

Motivazione IP Statico per il Server

Il server necessita di un indirizzo IP statico perché:

  • I servizi DNS e Active Directory devono essere raggiungibili sempre allo stesso indirizzo
  • I client devono conoscere l'IP del DNS server per la risoluzione nomi
  • La stabilità dell'indirizzo garantisce continuità dei servizi critici
  • Facilita la configurazione e il troubleshooting

Dominio Aziendale

Il dominio configurato è: wedev.local

Questa scelta garantisce:

  • Separazione logica dalla rete Internet pubblica
  • Gestione centralizzata di utenti e computer
  • Controllo unificato delle risorse condivise
  • Applicazione di policy di gruppo

Schema Architetturale

L'architettura di rete prevede:

  • Server centrale: eroga servizi AD DS, DNS, IIS e Desktop Remoto
  • Switch di rete: collega tutte le postazioni client al server
  • Router/Gateway: fornisce accesso Internet e routing tra rete locale e WAN
  • Client Windows: postazioni di lavoro unite al dominio wedev.local

4. IMPLEMENTAZIONE

L'implementazione dell'infrastruttura viene realizzata configurando un server Windows Server per fornire i servizi richiesti.

1. Configurazione di Rete del Server Windows

Sul server viene configurata manualmente la rete con IP statico.

Procedura:

  1. Pannello di controllo → Rete e Internet → Centro connessioni di rete
  2. Proprietà della scheda di rete → Protocollo IPv4:
    1. IP: 192.168.1.10
    2. Subnet mask: 255.255.255.0
    3. Gateway: 192.168.1.1
    4. DNS: 192.168.1.10
  3. Verificare con ipconfig e ping

Motivazione IP statico: Il server necessita di IP statico perché DNS e Active Directory devono essere sempre raggiungibili allo stesso indirizzo per garantire stabilità dei servizi.

2. Active Directory Domain Services – Domain Controller

Procedura:

  1. Server Manager → Aggiungi ruoli e funzionalità
  2. Selezionare Active Directory Domain Services
  3. Promuovere il server a Domain Controller
  4. Creare nuovo dominio: wedev.local
  5. Impostare livello funzionale e password ripristino
  6. Riavviare

Ruolo PDC: Gestisce autenticazione utenti, database Active Directory e coordina le operazioni di dominio.

Vantaggi AD: Gestione centralizzata, sicurezza migliorata, scalabilità, controllo unificato accessi.

3. DNS Server

Il DNS è essenziale per Active Directory per la risoluzione dei nomi nel dominio.

Configurazione:

  1. Il DNS viene installato automaticamente con AD DS
  2. Verificare zona diretta wedev.local in DNS Manager
  3. Controllare record automatici del Domain Controller

Tipo zona: Zona integrata in Active Directory con replica automatica.

Funzionamento: Il client interroga il DNS (192.168.1.10), che restituisce l'IP corrispondente al nome richiesto.

Esempio: Accesso a \\server\cartella → DNS risolve "server.wedev.local" in 192.168.1.10.

4. Gestione Utenti e Computer nel Dominio

Creazione OU:

  1. Utenti e computer di Active Directory
  2. Nuovo → Unità organizzativa: OU_Utenti, OU_Computer

Creazione utenti:

  1. In OU_Utenti → Nuovo → Utente
  2. Creare m.rossi e l.bianchi con password iniziale

Collegamento client:

  1. Client: Impostazioni → Sistema → Cambia nome computer o dominio
  2. Dominio: wedev.local
  3. Credenziali amministratore
  4. Riavvio

Accesso: Login con wedev\m.rossi

Vantaggi OU: Organizzazione logica e applicazione policy specifiche.

5. Web Server IIS

Installazione:

  1. Aggiungi ruoli → Server Web (IIS)
  2. Installa

Creazione sito:

  1. Gestione IIS → Aggiungi sito Web
  2. Nome: Intranet WeDev
  3. Percorso: C:\inetpub\wwwroot\intranet

Accesso: http://192.168.1.10 o http://server.wedev.local

6. Desktop Remoto e Condivisione Risorse

Desktop Remoto:

  1. Proprietà sistema → Connessione remota → Consenti connessioni
  2. Seleziona utenti autorizzati

Condivisione:

  1. Creare cartella (es. C:\Condivise\Progetti)
  2. Proprietà → Condivisione → Attiva condivisione
  3. Impostare permessi

Accesso: \\server\Progetti

5. LINGUAGGI E TECNOLOGIE UTILIZZATE

  • Windows Server 2019/2022
  • Active Directory Domain Services
  • DNS
  • IIS
  • Protocollo TCP/IP
  • RDP (Remote Desktop Protocol)

6. SVILUPPI FUTURI

In prospettiva futura, l'infrastruttura potrà essere migliorata attraverso:

L'introduzione di un server DHCP dedicato per gestire in modo più efficiente l'assegnazione degli indirizzi IP, riducendo conflitti e semplificando l'amministrazione.

L'implementazione di una VPN per accesso remoto sicuro, garantendo protezione delle comunicazioni esterne tramite cifratura del traffico dati.

L'aggiunta di backup automatici su cloud per proteggere i dati da guasti hardware ed errori, assicurando continuità operativa e rapido ripristino.

L'implementazione di policy di sicurezza avanzate con regole di accesso granulari, autenticazione robusta e meccanismi di monitoraggio.

Infine, la scalabilità dell'infrastruttura permetterà di supportare un numero maggiore di utenti senza compromettere le prestazioni.

7. CONCLUSIONI

Il progetto consente a WeDev S.r.l. di disporre di una rete aziendale moderna, sicura e scalabile.

Gli obiettivi iniziali sono stati raggiunti grazie alla centralizzazione dei servizi tramite Windows Server e Active Directory.

L'infrastruttura progettata garantisce gestione semplificata, sicurezza elevata e possibilità di espansione futura, rispondendo efficacemente alle necessità operative dell'azienda.
